方法一

步骤

1. 下载别人的项目源代码到本地。

从github上复制项目的https链接。

image.png

这里target-project-url是你想更改的项目在github上的https链接。

2. 删除该项目的默认git配置文件。

项目的源代码下载完成之后你需要在项目的根目录中删除一个名为.git的文件夹。
注意：该文件夹是隐藏文件，windows下可能会默认不显示，为了方便（作为程序员），这里用命令行进行删除。

target-project为你下载的项目名称，上面的命令的意思是进入该目录。

image.png

3.初始化git配置

删除了别人的github之后，你需要创建属于自己的git配置。

在项目根目录中运行这个命令会创建.git文件夹，这就是一个初始化的git配置。

4.github上创建新的代码仓库。

如果你是想把代码push到你的github上已有仓库中，这一个可以跳过，如果是想push到全新的仓库中，就需要在github上面创建新的代码仓库。

image.png

5.修改后的代码push到自己的指定repository中。

到这一步，本地代码只是提交到了本地仓库，还没有被push到远端仓库。

上面这一行代码就是本篇文章最最重要的一个命令，这是远端仓库的路径，设置完之后origin就会指向github远端仓库，以后代码就会push到这里。

最后push本地更改的代码到origin（远端仓库）。
这里 -u的作用就是你以后push代码的时候，可以直接用git push就可以把代码push到远端仓库 的mater分支中，方便了很多。

方法二

首先在GitHub上面创建新的仓库

然后克隆别人GitHub上面的项目到本地

clone项目

git克隆

克隆完成

然后进入刚刚克隆下来的 .git 文件夹里面改一下 config 的 url （改为刚刚创建的仓库路径），即可推送到自己的仓库

将你的代码上传到 Github 就可以了

git push –set-upstream origin master
